# CSV «Загрузка конверсий по ссылке» (Яндекс.Директ / Метрика)

Формат и требования: [официальная справка Директа](https://yandex.ru/support/direct/statistics/conversions.html).

Экспорт в проекте: `GET /export/yandex-direct-conversions.csv?token=…`

## Токен

Имя: **`DIRECT_CONVERSIONS_CSV_TOKEN`**.

Подставляется из **переменных окружения** процесса (Secrets на хостинге, Cursor и т.д.) и из **локального файла `.env`** в корне проекта (файл в `.gitignore`, в репозиторий не попадает). Порядок: сначала `_load_local_env()` подмешивает `.env`, затем в `settings` читается `os.environ`.

Ссылка: `/export/yandex-direct-conversions.csv?token=<то же значение, что в DIRECT_CONVERSIONS_CSV_TOKEN>`.

## Колонки (разделитель `;`, UTF-8)

| Колонка           | Обязательность | Описание |
|-------------------|----------------|----------|
| `create_date_time`| Да             | `dd.MM.yyyy HH:mm:ss` (не старше 113 дней) |
| `id`              | Нет            | ID сделки CRM или заявки |
| `client_uniq_id`  | Нет            | Стабильный ID клиента, у нас `deal_{id}` или `lead_{id}` |
| `client_ids`      | Нет            | ClientID Метрики (только положительное целое число) |
| `emails`          | Нет            | Email (пусто, если нет) |
| `phones`          | Нет            | Только цифры, с кодом страны (для РФ: `7…`) |
| `emails_md5`      | Нет            | MD5 email в нижнем регистре |
| `phones_md5`      | Нет            | MD5 от нормализованного телефона |
| `order_status`    | Да             | Только: `IN_PROGRESS`, `PAID`, `CANCELLED`, `SPAM` |
| `revenue`         | Нет            | Десятичная дробь с точкой, неотрицательная |
| `cost`            | Нет            | Оставляем пустым |

## Соответствие этапов CRM

| Этап сделки (тип воронки) | `order_status` | Ценность (руб.) |
|---------------------------|----------------|-----------------|
| В работе (`open`)         | `IN_PROGRESS`  | 300             |
| Успех (`won`)             | `PAID`         | 15000           |
| Отказ (`lost`)            | `CANCELLED` или `SPAM`* | 0       |

\*`SPAM`, если в причине отказа есть слова вроде «спам», «не целев».

Заявки с сайта без сделки выгружаются как `IN_PROGRESS` с ценностью 300.

## Цели Метрики

Счётчик: `87164937`.

В Метрике нужно создать JavaScript-события с такими идентификаторами:

| Идентификатор | Когда срабатывает |
|---------------|-------------------|
| `LEAD_SEND` | Любая успешная заявка с сайта |
| `LEAD_QUIZ` | Успешная заявка из квиза |
| `LEAD_CONTACT` | Успешная заявка из контактной формы |
| `LEAD_CALLBACK` | Успешная заявка из модалки обратного звонка |
| `LEAD_MEASURE` | Успешная заявка на замер |
| `DIRECT_STATUS_CHANGED` | Любая кнопка статуса из письма |
| `DIRECT_TARGET` | Кнопка «Целевой» из письма |
| `DIRECT_PAID` | Кнопка «Оплачен» из письма |
| `DIRECT_CANCELLED` | Кнопка «Нецелевой» из письма |
| `DIRECT_SPAM` | Кнопка «Спам» из письма |

CSV и цели работают вместе: сайт сразу отправляет цели заявок в Метрику, а CSV по ссылке отдаёт офлайн-статусы для Директа.
